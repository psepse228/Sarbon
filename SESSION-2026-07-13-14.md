# Cortège — сессия 2026-07-13/14

## 1. Диагностика и фикс схемы (первый живой запуск)

Первый локальный запуск дашборда (после Phases 2/3/4a) упал на `POST /api/test-chat`. Причина: Supabase-проект общий с другим ботом — CRM-ассистентом **Jonik's**, у которого уже была своя таблица `leads` (бизнесы/скрапинг, схема `business_name/category/city/...`). Миграция Cortège пыталась создать таблицу с тем же именем и молча схлопывалась.

**Фикс:** таблица Cortège переименована в `cortege_leads` везде — миграция `0005_add_leads.sql`, `backend/app/functions/handlers.py`, `dashboard/src/lib/leads.ts`. Таблица `leads` (Jonik's) не тронута. Заодно нашёлся и починен незамеченный баг: тестовые фикстуры в `test_handlers.py` ещё ссылались на старое имя `leads=`, из-за чего 2 теста молча "проходили" без реальной проверки merge-логики.

Также применено (уже подтверждено тобой в Supabase): создание `knowledge_gaps`, `cortege_leads`, колонки `disabled_skills`.

## 4. Расширение CRM-функций (merged `74bf9b9`)

Ты попросил перечитать конкурентный анализ (Business Robots AI) и добавить всё, что реально можно построить без платежей/новых интеграций. Собрано 8 задач:

1. **Мультиязычность** — бот отвечает на языке последнего сообщения гостя (раньше был жёстко зашит только русский)
2. **Kanban для лидов** (`/d/leads`) — 4 колонки (Новые/В работе/Забронировано/Потеряно), кнопки перемещения, включая восстановление из "Потеряно"
3. **Рассылки** (`/d/broadcasts`) — новая таблица `broadcasts`, backend `POST /internal/broadcast` (отправка через aiogram по списку chat_id, ошибки логируются, а не проглатываются), страница-композер + история
4. **Отзывы** (`/d/reviews`) — новая таблица `reviews`, новый always-on tool `capture_review` (бот фиксирует оценку, только если гость сам её дал — никогда не спрашивает первым), страница со звёздами и средним рейтингом
5. Два новых пункта в сайдбаре

**Осознанно НЕ делали:** оплата/депозит (нужно решить рынок — Узбекистан или США), WhatsApp/Instagram (нужны твои реальные API-ключи Meta), multi-tenant/команды (ты сам просил отложить до второго клиента), краулер базы знаний (отдельная большая задача).

## 5. Фон дашборда — живой шейдер GemSmoke (merged `44f364c`)

По твоему запросу заменил CSS-орбы на `@paper-design/shaders-react` — цвета Cortège (`#34d399` мятный, `#d9b872` золото, `#0b0d12` фон), `shape="none"`, медленная скорость. Отключается полностью при `prefers-reduced-motion`. Ты подтвердил: ставим на весь desktop-дашборд, не только на логин — с пониманием, что это живая WebGL-анимация тяжелее прежних CSS-орбов.

## 6. Побочная задача — лиды для холодного обзвона (не связано с Cortège)
